Chapter2 Parallelism

Work and Span

  • Work: The work of an algorithm corresponds to the total number of primitive operations performed by an algorithm.

    它告诉你这个算法如果你只有一个 processor，串行，它的时间复杂度是什么。

  • perfect speedup: If we had W work and P processors to work on it in parallel, then even division would give each processor W/P fraction of the work, and hence the total time would be W/P .We cannot break dependencies, we have to complete tasks in order.

  • Span: The span of an algorithm basically corresponds to the longest sequence of dependences in the computation. It can be thought of as the time an algorithm would take if we had an unlimited number of processors on an ideal machine.

    即最长的并行依赖链

  • The Work is the same both in Sequential composition and Parallel composition, however, the Span of Sequential composition is $1+S1+S2$ while $1+max(S1,S2)$ in Parallel composition.

  • According to the greedy scheduling principle, the range of time is $S≤T≤W/P+S$ .

    consider we have unlimited number of processors, 则 $lim_{P \to ∞} W/P+S=S$

    尝试证明：

    每个timestep 中可并行地执行n个任务，n取决于processer的个数以及任务之间的关系

    假设一共有N个timestep, 每个timestep执行的任务数为n_k，每次执行任务的时间都是单位时间Δt，在有p个processer的情况下:

    T_p <= Σ(k=1->N)[n_k/p]Δt Σ(k=1->N)n_k=W NΔt=S

    且[m/p] <= (m+p-1)/p

    故T_p <= Σ(k=1->N)[n_k/p]Δt <= (W-S)/P + S // 但怎么从这步到W/P+S呢

  • Work Efficiency: We say that a parallel algorithm is (asymptotically) work efficient, if the work is asymptotically the same as the time for an optimal sequential algorithm that solves the same problem.

  • In general the work is more important than the span.
